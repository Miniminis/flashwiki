# 라우팅

## 1. 라우팅과 라우터&#x20;

* 라우팅&#x20;
  * 네트워크에서 데이터(패킷)을 보낼 때, 최적의 경로를 찾는 과정
* 라우터&#x20;
  * 라우팅을 수행하는 장치&#x20;
  * 네트워크 사이에서 데이터를 전달하는 장치
  * 보통 둘 이상의 서로 다른 네트워크에 연결된다.&#x20;
  * 데이터(패킷)을 목적지로 보낼 때, 최적의 경로를 결정하고, 경로가 결정되면, 해당 경로로 데이터를 넘겨주는 라우팅을 수행한다.&#x20;
  * 라우팅 테이블을 기반으로 데이터를 다음 목적지로 전달한다.&#x20;

## 2. 라우팅 테이블

* IP 주소를 기반으로 라우터의 위치를 저장한 테이블 혹은 데이터베이스이다.&#x20;
* 다양한 네트워크에 대한 정보와 해당 네트워크에 연결하는 방법이 포함되어있다.&#x20;

### 2-1. 구성&#x20;

* 네트워크 대상 (Network Destination)&#x20;
  * 목적지 네트워크의 IP 주소&#x20;
* 서브넷 마스크 (Subnetmask)
  * 대상 주소를 설명할 때 쓰이는 값 &#x20;
* 게이트웨이 (Gateway)&#x20;
  * 이 장치와 연결되어있는 홉이자 패킷이 전달되는 다음 주소(외부 네트워크와 연결된 장치)&#x20;
  * 목적지가 로컬 네트워크라면 Connected(연결됨) 이라고 표기되고, 다른 네트워크라면 해당 네트워크의 게이트웨이를 가리킨다.&#x20;
* 인터페이스 (Interface)&#x20;
  * 게이트웨이로 가기 위해서 거치는 장치
  * 예를 들면, 10.0.0.2 는 eth3을 통해서 접근 가능하다. &#x20;
* 메트릭 (Matric)
  * 라우터가 라우팅을 할 때, 최적을 경로를 찾기 위해서 참고하는 값이다.
  * 우선순위라도고 불린다.&#x20;
  * 동일한 라우팅 테이블 요소가 2개 있을 때, 이 값이 낮은 요소가 선택된다.&#x20;
  * 일반적으로 홉의 수 (hop count), 지연시간, 처리량 등이 포함될 수 있다.&#x20;

## 3. 게이트웨이&#x20;

* 네트워크와 네트워크 사이를 잇는 장치&#x20;
* 프로토콜 변환기
* 하는 일은 라우터와 비슷하다.&#x20;

## 4. 홉&#x20;

* 네트워크에서 출발지와 목적지 사이에 위치한 장치를 의미한다.&#x20;
* 홉 카운트(hop count)데이터가 출발지부터 목적지까지 가는데 거치는 장치의 개수, 즉 홉의 개수를 말한다.&#x20;
* 홉의 개수가 적을수록 메트릭이 올라가 우선순위가 높아질 것이다.&#x20;
* 라우팅을 홉바이홉 통신이라고도 한다.  &#x20;
