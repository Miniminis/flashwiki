# 1. 쉽고 정확한 문장으로 문제해결

## 1. 짧게 쓰기&#x20;

### 1) 극 존칭

* 서비스를 종료하시겠습니까? -> 서비스를 종료할까요?&#x20;

### 2) 반복되는 말

* 제목과 동일한 단어부터&#x20;

### 3) 의미없는 정보

> 화면의 디자인은 다시 설명할 필요가 없다. \
> 예) 아래의 그림처럼\~&#x20;

* 앞으로 받을 배당금 -> 받을 배당금&#x20;
* 대출금의 원금을 갚습니다. -> 금액 입력하기&#x20;
* 발견한 인기 서비스를 써볼 수 있어요. -> 삭제&#x20;



## 2. 쉽게 쓰기&#x20;

### 1) 소리내서 말하듯&#x20;

> 잘 읽히지 않는다면 고쳐야할 문장이다.&#x20;

* 재 가입 의향이 있으실까요? -> 다시 만날 수 있을까요?&#x20;
* 찾으시는 상품으로 쇼핑 검색 -> 찾고 있는 상품 검색&#x20;
* 이 입력란을 작성하세요
* 아래의 이미지들 중 마음에 들지만 조금 부족한 이미지가 있다면 버튼을 눌러 다시 한번 생성해보세요.&#x20;
* 무료 요금제에서는 1일 7회 동영상 파일로 다운로드할 수 있습니다.&#x20;
  * 개선) 하루에 7번만 동영상 파일로 다운로드할 수 있어요!&#x20;
    * 버튼 : 클릭하여 무제한 다운로드하기
* 브랜드 일관성 유지를 위해 팀 관리자는 팀원에게 디자인 편집 및 에디터 메뉴에 대한 제한을 설정할 수 있습니다.&#x20;
  * 개선) 브랜드 디자인의 일관성 유지를 위해 에디터 메뉴를 제한할 수 있어요.&#x20;
* 구독, 결제 안내&#x20;

### 2) 한문, 영어 바꾸기&#x20;

* 인증번호 유효시간이 만료됐어요 -> 인증번호 입력시간이 지났어요&#x20;
* 은행명이 올바르지 않습니다. -> 은행 이름을 다시 확인해주세요.&#x20;
* 이번 주 브랜드 포커스 -> 이번주 눈에 띄는 브랜드&#x20;
* 계정 삭제시 내가 참여중인 워크스페이스 내 드라이브와 하위 항목이 모두 삭제됩니다. -> 모든 드라이브와 파일들을 삭제해요.&#x20;

### 3) 단어+단어 피하기&#x20;

> 단어가 3개 이상 나열되면 의미가 불명확해진다.&#x20;

* 세입금 정기분 할인 혜택 건당 150원 -> 납부할 때마다 최대 150원 할인&#x20;
* 내게 맞는 대출 찾기, 개인정보 조회 내역 안내 -> 내게 맞는 대출찾기에서 조회한 개인정보&#x20;

> 시점을 정확하게 써야 오해하지 않는다.&#x20;

* (제목) 카드값 미리내기 서비스가 종료돼요. : \~ 종료되었어요.&#x20;
  * (제목) 서비스 종료 안내 : \~ 종료됐어요.&#x20;
* 미성년자 송금 한도 증가&#x20;
  * \-> 000 님의 송금 한도가 늘었어요.&#x20;
* 작업 사이즈 변경 우려&#x20;
  * \-> 작업사이즈가 변경될 수 있어서 기능이 제공되지 않아요.&#x20;
* 원본 문서 복제 허용 -> 문서를 복제할 수 있어요&#x20;



## 3. 정확하게 쓰기&#x20;

### 1) 타이틀부터 제대로&#x20;

> 유저가 다음 상황을 예상할 수 있도록!&#x20;

* (안내) 일정 시간 사용하지 않는 경우, 서비스를 강제 종료합니다.&#x20;
  * \-> (다시 로그인 해주세요) 00 님의 개인정보를 보호하기 위해 자동으로 로그아웃 됐어요.&#x20;

### 2) 타이틀 + 버튼 조응하기&#x20;

> 유저가 버튼을 누르면 어떻게 되는지 직관적으로 알 수 있어야 한다.&#x20;

* 지금 돌아가면 처음부터 다시 해야해요. (취소/괜찮아요)&#x20;
  * \-> 다음에 할까요? (닫기/나가기)&#x20;
* 상담톡을 종료하시겠습니까? (취소/확인)&#x20;
  * 끝낼까요? (닫기/끝내기)&#x20;

### 3) 수동태 바꾸기&#x20;

> 유저의 행동과 그에 따른 결과가 더 명확해진다.&#x20;

* 알림 설정이 해제되었습니다. -> 알림 설정을 해제했어요.&#x20;
* 청약 신청이 되어있어요 -> 청약을 신청했어요.&#x20;
* 비밀번호가 일치하면, 000을 활성화 합니다. -> 000을 쓰려면 비밀번호를 눌러주세요.&#x20;

> 수동태를 능동태로 바꾸면 안되는 상황&#x20;

* 서비스를 종료할 때
  * 서비스가 종료돼요 -> OK&#x20;
  * 서비스를 종료해요 -> NO&#x20;
* 유저가 행동하지 않지만, 영향을 받는 것&#x20;
  * 연체 안내 - 연체돼요, 연체될 예정이에요.&#x20;

## 4. 기타&#x20;

## Case Study&#x20;

### 1. 고객이 고민하는 시간을 없에기&#x20;

* 주행 거리가 150,000km를 초과하나요?&#x20;
  * \-> 전손이력이 있나요?&#x20;

### 2. 명확한 가이드라인 주기&#x20;

* 이체 내역을 확인할 수 있는 계좌의 은행을 선택해주세요
  * 문제상황 : 이탈률이 높음&#x20;
  * 문제 정의 : 이체내역이 어렵게 느껴져 명확하게 이해하기가 어려움 &#x20;
  * 가설 설정 : 계좌번호를 모른다.&#x20;
  * 개선안 : 계좌번호를 알고 있는 은행을 선택해주세요.&#x20;

### 3. 쉽고 명확하게 설명하기&#x20;

* 해당 계좌는 비밀번호 사후 등록 계좌입니다. 먼저 은행 사이트에서 아이디와 비밀번호를 등록해주세요.&#x20;
  * 문제정의&#x20;
    * "사후 등록 계좌"라는 단어가 어렵다.&#x20;
    * 은행에서 비밀번호를 등록하지 않은 계좌라서 서비스 이용이 어렵다.&#x20;
  * 가설
    * 이 단계에서는 고객이 계좌를 이용하여 서비스를 등록하게 만드는 것이 가장 큰 목표인데, 비밀번호가 등록되지 않은 계좌는 어차피 서비스 등록을 할 가능성이 낮다. 따라서 다른 계좌로 서비스 이용을 유도하는 편이 나을 것이다.&#x20;
  * 개선
    * 비밀번호가 등록되지 않은 계좌에요. 다른 계좌로 진행하는 것은 어떨까요?&#x20;

### 4. 고객의 다양성을 존중하기&#x20;

* 최근 2년 내 대학을 졸업했나요?&#x20;
  * 문제 : 고객이 무조건 대학을 진학했을 것이라고 가정. 상대적 박탈감을 유도&#x20;
  * 가설 : 문제의 층위를 2개로 나눠 상대적 박탈감을 줄이고 명확한 답변 유도&#x20;
  * 개선
    * 대학을 졸업했나요?&#x20;
    * 졸업한 지 2년이 지났나요?&#x20;
